# GreedyARP

## Введение

Протокол **ARP (Address Resolution Protocol)** используется в локальных сетях для сопоставления IP-адресов с MAC-адресами. 
Когда устройство хочет узнать MAC по известному IP, оно отправляет **ARP-Request (broadcast)**, и владелец IP отвечает своим MAC-адресом.
Однако ARP не предусматривает механизмов аутентификации, что делает его уязвимым к атакам подмены (ARP spoofing / poisoning).

## Заголовки протоколов (псевдографика)

### Ethernet Frame
```
+---------------------+---------------------+---------------------+
| Destination MAC     | Source MAC          | EtherType           |
| (6 bytes)           | (6 bytes)           | (2 bytes)           |
+---------------------+---------------------+---------------------+
| Payload (e.g. ARP)                                          ...
+---------------------------------------------------------------+
```
### ARP Packet
```
+---------------------+---------------------+
| Hardware Type       | Protocol Type       |
| (2 bytes)           | (2 bytes)           |
+---------------------+---------------------+
| HW Size | Proto Size| Opcode              |
| (1 byte)| (1 byte)  | (2 bytes)           |
+---------------------+---------------------+
| Sender MAC Address (6 bytes)                |
+--------------------------------------------+
| Sender IP Address (4 bytes)                 |
+--------------------------------------------+
| Target MAC Address (6 bytes)                |
+--------------------------------------------+
| Target IP Address (4 bytes)                 |
+--------------------------------------------+
```

---

## Что делает программа
**GreedyARP** — это утилита, которая:
* перехватывает ARP-запросы в сети с помощью `libpcap`
* анализирует входящие пакеты
* отправляет **поддельные ARP-Relpy**, указывая **свой MAC-адрес** для любых запрашиваемых IP

## Предупреждение

Данная программа демонстрирует уязвимости ARP и предназначена **исключительно для образовательных целей**:
* изучение сетевых протоколов
* лабораторные работы
* тестирование безопасности в контролируемой среде
Использование в реальных сетях без разрешения может нарушать законы и правила эксплуатации сети.
